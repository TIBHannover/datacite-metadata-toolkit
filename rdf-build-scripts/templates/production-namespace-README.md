# DataCite TIB/W3ID Production Namespace

This directory is a generated production bundle for the TIB-maintained DataCite linked-data namespace. It is generated from `rdf-vocabulary-staging/` by running:

```bash
npm run build:production-namespace
```

Do not edit files in this bundle by hand. Make source changes in `rdf-vocabulary-staging/`, regenerate `production-namespace/`, review the generated diff, and then commit the regenerated bundle.

## Namespace

| Layer | URL |
|---|---|
| Persistent namespace | `{{CANONICAL_NAMESPACE}}` |
| GitHub Pages backend | `{{PUBLICATION_BASE_URL}}` |
| GitHub Pages path | `{{PAGES_BASE_PATH}}` |

All RDF identifiers in this bundle use the persistent W3ID namespace. The GitHub Pages backend serves the concrete files that W3ID redirects to.

## Contents

| Path | Purpose |
|---|---|
| `class/` | DataCite class definitions as JSON-LD plus a section index |
| `property/` | DataCite property definitions as JSON-LD plus a section index |
| `vocab/` | SKOS ConceptSchemes and controlled vocabulary terms plus a section index |
| `context/` | JSON-LD context files |
| `manifest/` | Versioned inventories and current-version pointers |
| `dist/` | Versioned and current distribution artifacts in JSON-LD, RDF/XML, Turtle, and OWL-related forms |

The generated section `index.html` files are for human browsing. The JSON-LD, Turtle, RDF/XML, manifest, and context files are the machine-readable namespace artifacts.

## Publication Flow

The intended flow is:

1. Source changes are made in `selgebali/datacite-metadata-toolkit`.
2. `production-namespace/` is regenerated and reviewed in that source repository.
3. The source repository is synced to `TIBHannover/datacite-metadata-toolkit`.
4. The TIB toolkit fork supplies the generated namespace files to `TIBHannover/datacite`.
5. `TIBHannover/datacite` serves the files through GitHub Pages, and W3ID redirects the persistent namespace to that Pages backend.

Publication-owned root files such as `.nojekyll`, `LICENSE`, and the root `index.html` belong in `TIBHannover/datacite`. They are not generated by this bundle unless the publication workflow explicitly chooses to manage them here.

## Checks

Before publishing, verify that:

- no machine-readable artifact in this bundle contains the staging namespace URL from the source tree
- representative files such as `class/Resource.jsonld`, `property/identifier.jsonld`, `vocab/resourceTypeGeneral/Dataset.jsonld`, `context/fullcontext.jsonld`, and `manifest/datacite-current.json` use `{{CANONICAL_NAMESPACE}}`
- W3ID redirects resolve to the corresponding files under `{{PUBLICATION_BASE_URL}}`
